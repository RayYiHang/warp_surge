# 🚀 Warp Account Manager - 快速启动指南

## ⚡ 问题解决

**问题**: `warp.local` 无法访问
**解决方案**: 使用本地HTTP服务器

## 🎯 三种启动方式

### 方式1: Python服务器 (推荐)

```bash
# 1. 进入项目目录
cd /Users/lyh/Downloads/warp.dev_account_manager

# 2. 启动服务器
python3 start-local-server.py

# 3. 访问管理界面
# 浏览器会自动打开 http://127.0.0.1:8080
```

### 方式2: Node.js服务器

```bash
# 1. 进入项目目录
cd /Users/lyh/Downloads/warp.dev_account_manager

# 2. 启动服务器
node start-local-server.js

# 3. 访问管理界面
# 浏览器会自动打开 http://127.0.0.1:8080
```

### 方式3: 直接HTML文件

```bash
# 1. 下载管理界面
curl -o warp-manager.html https://raw.githubusercontent.com/RayYiHang/warp_surge/refs/heads/main/web-manager.html

# 2. 双击打开HTML文件
open warp-manager.html
```

## 🌐 访问地址

启动成功后，你可以通过以下地址访问：

- **主要地址**: `http://127.0.0.1:8080`
- **备用地址**: `http://localhost:8080`

### 端口冲突解决方案

如果8080端口被占用，可以使用其他端口：

```bash
# Python
python3 start-local-server.py --port 8081

# Node.js
node start-local-server.js --port 8081
```

支持的端口选择：
- 8080 (默认)
- 8081
- 8082
- 8888
- 9000

## 🔧 使用流程

### 1. 启动本地服务器
```bash
# 选择一种方式启动
python3 start-local-server.py
# 或
node start-local-server.js
```

### 2. 获取账户数据
```
1. 登录 https://app.warp.dev
2. 按F12打开开发者工具
3. Application → Local Storage → app.warp.dev
4. 复制包含email和stsTokenManager的数据
```

### 3. 添加账户
```
1. 在管理界面粘贴数据
2. 点击"添加账户"
3. 命令自动复制到剪贴板
4. 在Surge控制台粘贴执行
```

### 4. 管理账户
```
- 查看账户列表
- 切换活跃账户
- 备份重要数据
```

## 🛠️ Surge控制台操作

### 打开Surge控制台
1. 打开Surge应用
2. 设置 → 脚本编辑器
3. 粘贴生成的命令
4. 点击执行

### 常用命令参考
```javascript
// 账户管理
accountManager.getAccountList()
accountManager.setActiveAccount("your-email@example.com")
accountManager.getActiveAccount()

// 数据管理
persistenceManager.backupData()
persistenceManager.getStorageStats()
persistenceManager.clearAllData()
```

## 🚨 故障排除

### 端口被占用
```bash
# 查看端口占用
lsof -i :8080

# 使用其他端口
python3 start-local-server.py --port 8081
```

### Python/Node.js未安装
- **Python**: 从 https://python.org 下载安装
- **Node.js**: 从 https://nodejs.org 下载安装

### 浏览器无法访问
1. 检查服务器是否启动成功
2. 确认端口号正确
3. 尝试 `http://localhost:8080`
4. 检查防火墙设置

### Surge命令未执行
1. 确认Surge模块已启用
2. 重启Surge应用
3. 检查命令格式是否正确
4. 查看Surge错误日志

## 📊 优势对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **本地服务器** | ✅ 像网站一样访问<br>✅ 自动更新<br>✅ 无需文件管理 | 需要启动服务器 | 推荐使用 |

## 💡 使用建议

1. **首次使用**: 推荐Python服务器方式
2. **临时使用**: 可以直接使用HTML文件
3. **高级用户**: 直接在Surge控制台执行命令
4. **定期备份**: 每周备份一次账户数据
5. **安全使用**: 不要在公共电脑使用
